# 个人信息库使用说明

## 概述

`personalInfo.ts` 文件用于将你的个人信息格式化为 AI 友好的格式，以便 AI 助手能够准确回答关于你的问题。

## 数据结构

个人信息库包含以下主要部分：

### 1. 基本信息 (basic)
- 姓名（中英文）
- 个人简介（中英文）

### 2. 教育背景 (education)
- 学校名称
- 学位
- 时间范围
- GPA
- 导师信息
- 核心课程列表

### 3. 工作经历 (experience)
- 实习/工作公司
- 职位
- 时间范围
- 主要职责列表

### 4. 项目经历 (projects)
- 项目名称
- 时间范围
- 项目描述列表

### 5. 获奖经历 (awards)
- 奖项名称
- 获奖情况
- 相关项目
- 个人贡献

### 6. 技能 (skills)
- 编程语言
- 框架/库
- 工具
- 平台

## 如何更新个人信息

### 方法 1：通过 locales.ts 自动生成（推荐）

个人信息库会自动从 `locales.ts` 文件中提取信息。要更新个人信息：

1. 编辑 `src/locales.ts` 文件
2. 更新对应语言（中文 `zh` 或英文 `en`）的 `experience` 部分
3. 个人信息库会自动同步更新

### 方法 2：直接修改 personalInfo.ts

如果需要添加 `locales.ts` 中没有的信息（如技能列表），可以直接编辑 `getPersonalInfo` 函数中的 `skills` 部分：

```typescript
skills: {
  languages: ['TypeScript', 'JavaScript', 'Python', 'Java', 'C++'],
  frameworks: ['React', 'Next.js', 'Vue', 'Node.js'],
  tools: ['Figma', 'Git', 'VSCode', 'Hugging Face', 'Transformers'],
  platforms: ['AWS', 'OpenAPI', 'VSCode Extension'],
},
```

## 系统提示词格式

`formatPersonalInfoForAI` 函数会将个人信息格式化为结构化的系统提示词，包含：

1. **清晰的章节划分**：使用 Markdown 格式，便于 AI 理解
2. **结构化信息**：每个部分都有明确的字段和格式
3. **回答要求**：指导 AI 如何基于这些信息回答问题

### 提示词特点

- **准确性**：只基于提供的信息回答，不编造
- **专业性**：回答要专业、友好
- **语言支持**：支持中英文两种语言
- **简洁性**：回答要简洁明了，重点突出

## 最佳实践

1. **保持信息更新**：定期更新个人信息，确保 AI 回答的准确性
2. **结构化数据**：使用列表和清晰的格式，便于 AI 解析
3. **完整性**：尽量包含所有相关信息，但避免冗余
4. **语言一致性**：确保中英文信息对应一致

## 示例

当你询问 "你在字节跳动做什么工作？" 时，AI 会基于以下信息回答：

```
## 工作经历
### 实习经历
- 公司：字节跳动
- 职位：软件工程师实习生，AI 平台
- 时间：2025年6月 - 至今
- 主要职责：
  - 领导了将 HTML 设计和 OpenAPI 规范转换为像素级完美 React 代码的规则驱动架构。
  - 为生成的应用程序实现了编译和运行时错误的自修复功能。
  ...
```

AI 会基于这些信息生成准确、专业的回答。
